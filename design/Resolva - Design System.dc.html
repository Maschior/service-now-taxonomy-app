A linguagem visual do Resolva

Um conjunto de decisões reutilizáveis — cores, tipografia, espaçamento e componentes — para que toda tela de encerramento de chamado seja consistente, rápida de ler e à prova de erro. Feito para analistas de suporte que fecham dezenas de tickets por dia.

📌 Conceito-chave: token

Um token é uma decisão de design com nome. Em vez de espalhar #FFFFFF por 200 telas, você o chama de --bg-surface. É exatamente por isso que o dark mode funciona: o token aponta para branco no tema claro e cinza-escuro no escuro — a tela inteira muda sem tocar em um componente sequer.

/* claro */ --bg-surface: #FFFFFF;
/* escuro */ --bg-surface: #1C1C21;
// o card só diz: background: var(--bg-surface)
Base

Princípios

Quatro regras que orientam toda decisão. Quando estiver em dúvida sobre um componente, volte aqui.

01
Clareza acima de estética

O analista precisa entender o estado do chamado em um relance. Legibilidade vem antes de qualquer enfeite.

02
Consistência reduz erro

O botão "Encerrar" é sempre igual, no mesmo lugar. Padrão repetido vira memória muscular.

03
Menos é mais

Muito espaço em branco, pouca cor. O vermelho só aparece onde há ação ou atenção — nunca decorando.

04
Estados explícitos

Todo componente mostra claramente: normal, foco, erro, desabilitado. O sistema nunca deixa o usuário no escuro.

Espaçamento & grid

Tudo é múltiplo de 4px. Uma base única faz os elementos "se encaixarem" sem ajuste manual. Use a escala abaixo para padding, gaps e margens — nunca um valor solto como 7px ou 13px.

Fundamentos

Bordas & sombras

Cantos suaves e sombras discretas. Num produto minimalista a hierarquia vem da borda e do fundo, não de sombras pesadas — elas só aparecem em elementos que "flutuam" (menus, modais). No tema escuro a sombra fica mais densa para compensar o contraste reduzido.
